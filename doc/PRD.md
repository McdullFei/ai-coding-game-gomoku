# 五子棋游戏产品设计文档（PRD）

## 1. 项目概述

### 1.1 项目背景

五子棋（Gomoku）是一款经典的策略棋类游戏，深受全球玩家喜爱。本项目旨在开发一款功能完善的人机对战与双人对战五子棋游戏，包含AI难度分级和玩家等级成长系统。

### 1.2 项目目标

- 实现完整的五子棋游戏核心玩法
- 提供三种AI难度级别（简单、中级、困难）
- 支持双人对战模式
- 引入等级成长系统，激励玩家提升段位

---

## 2. 五子棋基本规则

### 2.1 棋盘与棋子

- **棋盘尺寸**：15×15（225个交叉点）
- **棋子**：黑棋与白棋
- **落子位置**：棋盘交叉点，每次只能落一子

### 2.2 行棋规则

1. **落子顺序**：黑棋先行，之后黑白交替落子
2. **禁着点**：棋盘上已有棋子的交叉点不可落子
3. **游戏结束**：当一方在横、竖、斜（含对角线）任意方向上连成5子及以上时，该方获胜

### 2.3 胜负判定

- 率先连成5子（或以上）的一方获胜
- 棋盘下满且无5连珠时为平局

## 3. 功能需求

### 3.1 游戏模式

#### 3.1.1 人机对战模式

- 玩家执黑棋（先行）
- 电脑执白棋（后行）
- 电脑AI分为三个难度级别：
  - **简单（Easy）**：随机落子或仅有基础防守
  - **中等（Medium）**：具备一定攻守能力，能识别简单连四
  - **困难（Hard）**：深度算法，能识别连五、活四、冲四、做棋等

#### 3.1.2 双人对战模式

- 玩家1执黑棋（先行）
- 玩家2执白棋（后行）
- 轮流落子，通过同一设备操作

### 3.2 等级成长系统

#### 3.2.1 等级段位

| 段位 | 等级范围 | 胜利经验值要求 |
|------|----------|----------------|
| 新手 | 1-10 | 0 |
| 初级 | 11-20 | 100 |
| 中级 | 21-30 | 500 |
| 高级 | 31-40 | 1500 |
| 专家 | 41-50 | 4000 |
| 大师 | 51+ | 10000 |

#### 3.2.2 经验值获取规则

- **击败简单AI**：+10经验值
- **击败中级AI**：+25经验值
- **击败困难AI**：+50经验值
- **失败（任意难度）**：+5经验值（参与奖）
- **平局**：+3经验值

#### 3.2.3 等级显示

- 界面显示当前段位名称和等级
- 胜利后显示经验值变化动画

#### 3.2.4 规则说明

- 双人对战模式不参与等级成长
- 经验值仅在人与AI对战时计算

### 3.3 用户交互流程

1. **主菜单**
   - 开始游戏
   - 选择游戏模式（人机/双人对战）
   - 选择AI难度（如为人机模式）
   - 查看当前等级

2. **游戏中**
   - 显示当前回合（黑方/白方）
   - 显示上次落子位置高亮
   - 悔棋功能（可选）
   - 认输功能
   - 重新开始

3. **游戏结束**
   - 显示胜者
   - 显示经验值变化（如有人机对战）
   - 返回主菜单 / 再来一局

## 4. 技术需求

### 4.1 平台

- Web应用（浏览器端运行）
- 响应式设计，支持PC和移动端

### 4.2 AI算法

- **简单**：随机选择可落子点，概率性防守
- **中等**：Minimax算法，搜索深度2-3层
- **困难**：Minimax + Alpha-Beta剪枝，搜索深度4-6层，评估函数包含连五、活四、冲四、活三等棋型

### 4.3 数据存储

- 玩家等级和经验值存储在本地（localStorage）
- 无需后端服务器

## 5. UI/UX设计

### 5.1 视觉风格

- 简约现代中国风
- 棋盘使用木质纹理或淡雅配色
- 黑白棋子使用经典圆形设计，带阴影立体感

### 5.2 颜色方案

- 背景：米白色或淡木色
- 棋盘：浅黄色或木纹色
- 黑棋：深黑色
- 白棋：乳白色
- 选中高亮：淡蓝色
- 胜利连线：高亮显示

### 5.3 界面布局

- 顶部：游戏标题 + 当前等级显示
- 中部：15×15棋盘
- 底部：操作按钮（重新开始、悔棋、认输、返回菜单）
- 模式选择：弹窗或独立页面

## 7. 非功能性需求

- **性能**：落子响应时间 < 100ms（简单/中等难度），困难难度AI思考时间 < 3秒
- **兼容性**：支持Chrome、Safari、Firefox、Edge最新版本
- **可访问性**：支持键盘操作
